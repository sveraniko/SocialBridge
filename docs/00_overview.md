# 00_overview — SocialBridge

## 1) Контекст
SIS уже “боевой” и трогать его лишний раз нельзя: цена ошибки высокая.  
При этом нужен внешний слой для **автоматизации продаж из соцсетей** (Instagram/TikTok) без ручных переписок.

Мы используем ManyChat как “входной шлюз” (триггеры, автоответы), а SocialBridge как наш независимый мозг и трекер.

---

## 2) Цели (что считается успехом)
1) **Из комментария/сторис/DM → 1 клик → карточка товара/лука в SIS**
2) Никакого ручного “скинь ссылку/напиши менеджеру”
3) Трекинг кликов и конверсий на уровне shortlink (даже если SIS не умеет campaign)
4) Стабильность: падение/изменение соцсетей не должно ломать SIS

**Минимальный успех (MVP):**
- Instagram через ManyChat работает end-to-end:
  - Trigger → resolve → shortlink → deeplink → SIS открывает нужный объект

---

## 3) Негативные цели (что мы НЕ делаем)
- Не строим “супер-интеграцию” напрямую с Instagram/TikTok API в v0.
- Не добавляем новый функционал в SIS.
- Не пытаемся заменить ManyChat (пусть они страдают с комплаенсом и партнёрством).

---

## 4) Ключевые решения
### 4.1 Отдельный сервис
SocialBridge живёт отдельным репозиторием и деплоится независимо от SIS.

**Причина:** SIS большой; любые “мелкие правки” рискованны, и разработка превращается в мясорубку.

### 4.2 Shortlink как единая точка правды
Все сообщения ведут на `go.DOMAIN/t/{slug}`:
- можно менять маршрутизацию без переупаковки flows
- можно считать клики (и дальше, возможно, покупки)

### 4.3 Маппинг вместо магии
Для “пост → товар” используем `content_map`, а не LLM/парсинг “по ощущениям”.  
LLM можно добавить позже как fallback, но MVP должен быть железный и предсказуемый.

---

## 5) Пользовательские сценарии (user stories)
1) **Пользователь отвечает на сторис** → получает ссылку на конкретный товар (SIS)
2) **Пользователь пишет ключевое слово** (например “SIZE”) → получает ссылку на размерный сценарий или конкретный лук
3) **Пользователь комментирует пост** → получает DM со ссылкой на товар из этого поста
4) **Ничего не сматчилось** → получает ссылку на каталог + подсказку

---

## 6) Ограничения и риски
### 6.1 Гео и правила платформ
TikTok может быть недоступен для подключения через ManyChat в регионе.  
**Митигируем:** канал-агностичная архитектура и режим “TikTok → bio link → shortlink → SIS”.

### 6.2 Telegram лимиты start_param
Ограничение 64 символа на `start`.  
**Митигируем:** payload короткий, коды/LOOK_ фиксированной длины, никакого JSON в payload.

### 6.3 GDPR / хранение данных
Хранить минимум. По умолчанию не хранить IP; если надо антифрод — только хеши с солью + короткий retention.

---

## 7) Метрики успеха
- Resolve success rate ≥ 90% (на тех flow, где есть маппинг)
- CTR ≥ 15–30% по “горячим” триггерам (комменты на продуктовых постах)
- Нулевая деградация SIS (никаких релизов SIS ради SocialBridge)

---
